## 1. What Is a Service?

A **service** is a piece of functionality that can be used by other systems over a network.

### Key Points
- A service is what you connect together using **Web Services**
- A service is the **endpoint** of a connection
- Each service performs a **specific task**

### Functionalities of a Service
A good service should be:

- **Well-defined**  
  â†’ It clearly describes what it does
- **Self-contained**  
  â†’ It can work independently
- **Independent of other services**  
  â†’ It does not depend on the state or context of other services

 **Example:**  
A weather service that takes a city name and returns temperature.

---

## 2. What Is a Web Service?

A **web service** is a service that is available over the internet and can be accessed by other applications.

### Definition
- Web services are **self-describing**
- They are **stateless** (do not remember previous requests)
- They perform **small, discrete units of work**
- They are accessible over a **network**

### Web Service Providers
- Providers offer **APIs**
- Developers use these APIs to access functionality
- No need to install full applications

 **Example:**  
Google Maps API, Payment Gateway API, Weather API

---

## 3. Web Services Description Language (WSDL)

**WSDL** describes how to use a web service.

### Key Features
- Written in **XML**
- Defines:
  - Data types
  - Messages
  - Communication rules

### Types of Operations

1. **One-way**  
   - Message is sent
   - No response required

2. **Request and Response**  
   - Request is sent
   - Response is received

3. **Solicit Response**  
   - Service requests a response from the client

4. **Notification**  
   - Message sent to multiple receivers

---

## 4. Service Oriented Architecture (SOA)

### Definition
**Service Oriented Architecture (SOA)** is a system design where multiple services communicate with each other.

### Characteristics
- Collection of independent services
- Services communicate over a network
- Loosely coupled architecture
- Services can be reused across different systems

### Purpose of SOA
- Flexibility
- Easy integration
- Reusability
- Scalability

 **Example:**  
An online shopping system:
- Login service
- Payment service
- Order service
- Delivery service

### Implementation
- Usually implemented using the **Web Service model**

---

## 5. Quality of Service (QoS)

### Original Definition
Quality of Service (QoS) refers to technologies used to manage network traffic efficiently to improve user experience.

### Modern Usage
QoS is now a broader term used for:
- Customer care evaluations
- Technological evaluations

---

## 6. QoS in Customer Care Evaluations

- Focuses on **customer experience**
- Measured by issues that negatively affect users
- Only problems affecting service quality are evaluated

### Examples
- Service downtime
- Slow response
- Failed transactions

---

## 7. QoS in Technological Evaluations

- Focuses on **system efficiency**
- Ensures optimal use of resources
- May involve:
  - Improving software
  - Adjusting procedures
  - Optimizing code

---

## 8. Service Level Agreement (SLA)

### Definition
A **Service Level Agreement (SLA)** is a formal contract between a service provider and a customer.

- Defines services in **measurable terms**
- Uses **QoS metrics**

### Common Contents of an SLA

#### Performance Metrics
- Up-time and down-time ratio
- System throughput
- Response time

#### Problem Management
- Issue reporting process
- Resolution time

#### Penalties
- Compensation for service failure

#### Security
- Documented security measures
- Data protection policies

 **Example:**  
If a cloud service does not meet the agreed uptime, penalties are applied.

## 1. Scalability

### What is Scalability?

**Scalability** is a desirable property of a system, network, or process that shows its ability to:

- Handle **growing amounts of work** smoothly  
- Be **expanded or enlarged easily** when required  
- Maintain performance as demand increases  

In simple words, scalability means:
> *â€œCan the system grow when more users or workload come?â€*

 **Example:**  
A website that works fine for 100 users and can also handle 10,000 users by adding more resources.

---

## 2. Elasticity

### What is Elasticity?

**Elasticity** refers to the ability of a system to:

- **Increase resources when demand increases**
- **Reduce resources when demand decreases**
- Automatically return to its original state after scaling

The term comes from physics:
> *An elastic body returns to its original shape after being stretched or compressed.*

 **Example:**  
Cloud servers automatically increase during peak traffic and decrease at night.

---

## 3. How to Achieve Scalability & Elasticity?

Scalability and elasticity are mainly achieved through:

- **Dynamic Provisioning**
- **Multi-tenant Design**

---

## 4. Dynamic Provisioning

### What is Dynamic Provisioning?

**Dynamic Provisioning** is a method where computing resources (servers, storage, networks) are:

- Automatically **provisioned (added)**
- Automatically **de-provisioned (removed)**
- Controlled through an **administrative console or client application**

These actions can be performed by:
- Server administrator
- Network administrator
- Any authorized user

 In short:
> Resources are allocated **on demand**, not in advance.

---

### Problems in Traditional Computing Model

In traditional systems, two major problems exist:

#### 1. Underestimation of System Utilization
- Results in **under-provisioning**
- System becomes slow or unavailable during high demand
- Lost users and revenue

#### 2. Overestimation of System Utilization
- Results in **over-provisioning**
- Resources remain unused
- Leads to **low utilization and high cost**

---

### Why Cloud Resources Should Be Provisioned Dynamically?

Cloud computing uses dynamic provisioning to handle:

- **Seasonal demand variations**  
    Example: Online shopping during sales

- **Demand variations between different industries**  
    Example: Education vs finance workloads

- **Burst demand during extraordinary events**  
    Example: Live streaming of major events, exams, or launches

---

## 5. Multi-tenant Design

### What is Multi-tenant Design?

**Multi-tenant design** is a software architecture principle where:

- A **single instance** of software runs on a server
- It **serves multiple client organizations (tenants)**
- Each tenant shares the same infrastructure

---

### How Multi-tenancy Works

- Data and configuration are **virtually partitioned**
- Each client gets:
  - Separate data
  - Custom configuration
- Clients experience it as their **own virtual application**

  **Example:**  
Google Workspace serving multiple companies using the same platform.

---

## 6. Client-Oriented Requirements in Multi-tenant Design

### 1. Customization

- Multi-tenant applications must support a **high level of customization**
- Each organization may need:
  - Different settings
  - Different workflows
  - Different user roles

---

### 2. Quality of Service (QoS)

Multi-tenant applications must ensure:

- **Security**  
  - Data isolation between tenants

- **Robustness**  
  - One tenantâ€™s issue should not affect others

- **Reliable performance**  
  - Fair resource distribution among clients

---

## 7. Summary

- **Scalability** â†’ Ability to grow with increased workload
- **Elasticity** â†’ Ability to scale up and scale down automatically
- **Dynamic Provisioning** â†’ On-demand allocation of resources
- **Multi-tenant Design** â†’ Single software instance serving multiple clients

# Availability, Reliability, Fault Tolerance, Resilience & Security

---

## 1. Availability

### What is Availability?

**Availability** is the degree to which a system, subsystem, or equipment is:

- In an **operable and usable state**
- Ready **when it is needed**
- Available at an **unknown time**

In simple words:
> *Availability means how often the system is up and running.*

### Cloud Systems and Availability
- Cloud systems usually require **very high availability**
- Downtime must be minimal

 **Example:**  
A **â€œFive Ninesâ€** system provides:
- **99.999% availability**
- Only a few minutes of downtime per year

---

## 2. Reliability

### What is Reliability?

**Reliability** is the ability of a system or component to:

- Perform its **required functions**
- Under stated conditions
- For a **specified period of time**

In simple words:
> *Reliability means how long a system works correctly without failure.*

---

## 3. How to Achieve Availability & Reliability?

These properties are achieved through:

- **Fault-tolerant systems**
- **System resilience**
- **Reliable system security**

---

## 4. Fault Tolerance

### What is a Fault-Tolerant System?

**Fault tolerance** is the property that allows a system to:

- Continue operating correctly
- Even when **one or more components fail**

If performance decreases:
- The decrease is **proportional to the failure severity**
- Unlike poorly designed systems where a small fault causes total failure

 **Example:**  
If one server fails, another server immediately takes over.

---

## 5. Characteristics of a Fault-Tolerant System

A fault-tolerant system has **four basic characteristics**:

### 1. No Single Point of Failure (SPOF)

- No single component failure should bring down the entire system

### 2. Fault Detection and Isolation

- System must detect a fault
- Identify the faulty component

### 3. Fault Containment

- Prevent fault from spreading to other components

### 4. Availability of Reversion Modes

- System maintains **checkpoints**
- Allows rollback to a safe state

---

## 6. Single Point of Failure (SPOF)

### What is SPOF?

A **Single Point of Failure** is:

- A component which, if it fails,
- Causes the **entire system to stop**

Identifying SPOFs helps:
- Find critical components
- Prevent total system failure

---

### Preventing Single Point of Failure

- System must continue to operate
- Even **during repair or maintenance**
- Achieved using redundancy

 **Example:**  
Multiple servers instead of one server.

---

## 7. Fault Detection and Isolation (FDI)

### What is FDI?

**Fault Detection and Isolation** is a subfield of control engineering that:

- Monitors system behavior
- Detects when a fault occurs
- Identifies:
  - Type of fault
  - Location of fault

---

### Isolating the Failing Component

- When failure occurs:
  - Faulty component is isolated
  - Rest of the system continues working

---

## 8. Fault Containment

### What is Fault Containment?

- Some failures spread and affect the whole system
- Fault containment mechanisms:
  - Isolate faulty components
  - Protect healthy parts of the system

 **Example:**  
Blocking a malfunctioning node from sending incorrect data.

---

## 9. Availability of Reversion Modes

- System maintains **checkpoints**
- Used to manage system state changes
- Helps recover to a known safe state after failure

---

## 10. System Resilience

### What is Resilience?

**Resilience** is the ability of a system to:

- Maintain an **acceptable level of service**
- During faults or abnormal conditions
- Quickly return to normal operation

In simple words:
> *Resilience means how fast a system recovers after failure.*

---

### Examples of Risk Events

- Power loss at a plant for several days
- Loss of key services due to database corruption

Key question:
> *Can the system recover and resume normal operation?*

---

## 11. Disaster Recovery

### What is Disaster Recovery?

**Disaster Recovery (DR)** includes:

- Processes

Used to:
- Recover or continue IT infrastructure
- After natural or human-made disasters

---

### Common Disaster Recovery Strategies

- Replicate the entire system

#### Preparing
- Local mirror systems
- Surge protectors
- Uninterruptible Power Supply (UPS)

### 2. Identity Management

> *Manageability means how easily a cloud system can be controlled, maintained, and administered.*

## 2. Interoperability

### What is Interoperability?

**Interoperability** is a property of a system or product whose:

- Interfaces are **fully understood**
- Can work with **other systems or products**
- Supports **present and future systems**
- Does not require restricted access or special implementation

In simple words:
> *Interoperability means different systems can work together smoothly.*

### Control Loops in Autonomic Components

#### 2. Global Control Loop
- Coordinates with other components

### Key Elements of an Autonomic Component

- **Sensors**
  - Used for self-monitoring
- **Effectors**
  - Used for self-adjustment
- **Knowledge**
  - Stores policies and system information
- **Planner / Adapter**
  - Applies policies based on:
    - Self-awareness

### 4. Self-Protection

# Performance & Optimization in Cloud Computing

---

## 1. Performance Guarantees

### What are Performance Guarantees?

In cloud computing, **performance guarantees** mean that:

- Applications running in the cloud should deliver **expected performance**
- Users should experience:
  - Fast response time
  - Smooth execution
  - Reliable throughput

Because cloud platforms provide **huge computing power**, application performance must be guaranteed.

Cloud providers:

- Use **powerful infrastructure**

  - High-performance servers
  - Storage systems
  - Networks
- Build a **highly optimized environment**

Performance and optimization in cloud systems are achieved using:

---

## 3. Parallel Processing

### What is Parallel Processing?

**Parallel processing** is a computing technique where:

- Multiple calculations are performed **simultaneously**
- A large problem is divided into **smaller sub-problems**
- Each sub-problem is solved **at the same time**

In simple words:
> *Work is done faster by doing many things at once.*

ðŸ“Œ **Example:**  
Big data processing where data is split and processed across multiple machines.

## 4. Load Balancing

### What is Load Balancing?

**Load balancing** is a technique used to:

- Distribute workload evenly across:
  - Computers
  - Servers
  - CPUs
  - Network links
  - Storage devices
- Prevent any single resource from being overloaded

---

### Objectives of Load Balancing

Load balancing helps to:

- Achieve **optimal resource utilization**
- Maximize **throughput**
- Minimize **response time**
- Avoid **system overload**

---

### Why Should Systems Be Load Balanced?

- Improve overall **resource utilization**
- Improve **system performance**
- Improve **energy efficiency**

ðŸ“Œ **Unbalanced System:**  
Some servers overloaded while others are idle.

---

## 5. Job Scheduling

### What is a Job Scheduler?

A **job scheduler** is a software application responsible for:

- Managing unattended background executions
- Handling batch processing jobs
- Automatically executing tasks without human intervention

---

### What Should Be Scheduled in the Cloud?

Cloud job scheduling includes:

- **Computation-intensive tasks**
- **Dynamically growing and shrinking tasks**
- Tasks with **complex processing dependencies**

---

### How to Approach Job Scheduling?

Job scheduling can be implemented using:

- **Pre-defined workflows**
  - Tasks follow a fixed execution sequence

- **System automatic configuration**
  - System dynamically adapts based on workload
